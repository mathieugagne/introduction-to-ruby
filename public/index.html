A little history

A dynamic, open source programming language with a focus on simplicity and productivity. It has an elegant syntax that is natural to read and easy to write.

  • Creator: Yukihiro “Matz” Matsumoto

Similarities to C++

  • Ruby itself is written in C
  • Object-Oriented Design
  • public, private, and protected do similar jobs.
  • Operators are the same except Ruby doesn’t have ++ or --
  • You can also have constants, though there’s no special const keyword
  • Inheritance syntax is still only one character, but it’s < instead of :
  • You may put your code into “modules”, similar to how namespace in C++ is used
  • Exceptions work in a similar manner

Differences to C++

See a more comprehensive list here.

  • When tested for truth, only false and nil evaluate to a false value. Everything else is true (including 0, 0.0, and "0")
  • It’s self instead of this
  • Single inheritance only, on purpose

Methods

Access Levels: public, protected, private

  • Defaults to public
  • protected methods can be called by any instance of the defining class or its subclasses
  • private methods can be called only from within the calling object
  • You cannot access another instance's private methods directly.
  • protected vs private, follow example with dwarf.rb

attr_ .. helpers

  • attr_reader: creates a method to read the associated instance variable
  • attr_writer: creates a method to write to the associated instance variable
  • attr_accessor: adds both attr_reader & attr_writer methods

class Pony
  attr_accessor :name

  # same as
  attr_reader :name
  attr_writer :name

  # same as
  def name
    @name
  end
  def name=(value)
    @name = value
  end
end

Variables

  • name could be a local variable, scoped to its immediate definition
  • @name is an instance variable, scoped to the instance
  • $name is a global variable, accessable from anywhere.
  • It is not recommended to use global variables. They make programs cryptic.

Pseudo Variables

  • self: The receiver object of the current method.
  • true: Value representing true.
  • false: Value representing false.
  • nil: Value representing undefined.
  • __FILE__: The name of the current source file.
  • __LINE__: The current line number in the source file.

Symbol

http://www.ruby-doc.org/core-2.1.1/Symbol.html

Simply a name used in your program

  • works just like a variable
  • it is immutable after initialization
  • often use in Hash
  • Identical symbols are a single internal object
  • == tests for object_id equality
  • Symbols are not strings, do not have the same methods
  • Cannot be concatenated
  • Far more memory efficient than String (40%)
  • Symbol are not marked for desctruction by Garbage Collector
  • Watch for memory leaks!

Block

You can iterate over the elements in an array using Array.each, which takes a block.

A block is a chunk of code that lives inside a control statement, loop, method definition, or method call. It returns the value of its last line. In Ruby, blocks can be created two ways: with braces or with a do/end statement.


5.times { print "Odelay!" }
# or
5.times do
  print "Odelay!"
end
# Try this:
[1,2,3,4].each { |number| puts number }

Iterator & Loops

  • while: executes code while condition is true
  • until: executes code while condition is false
  • break: terminates the most internal loop
  • next: jumps to next iteration of the most internal loop
  • redo: restarts this iteration of the most internal loop, without checking loop condition.
  • for i in 1..5: Executes code once for each element in expression
  • [1,2,3,4].each { |number| puts number }: Same as above. Preferred version
  • The for loop is used to iterate an object. The Ruby .each method is preferred over the for loop because the for loop does not create a new scope for the object whereas the .each method does. The for loop is rare in Ruby.

Struct.new

A Struct is a convenient way to bundle a number of attributes together, using accessor methods, without having to write an explicit class


Customer = Struct.new(:name, :address) do
  def greeting
    "Hello #{name}!"
  end
